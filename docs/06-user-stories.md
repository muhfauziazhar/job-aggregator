## How to Read This Doc

- Stories grouped by **Epic**.
- Each story has an ID like `STORY-NN` (replace `STORY` with your project prefix, e.g. `FYP-01`).
- Each story is later promoted to one **GitHub Issue**, assigned to the milestone defined in `07-roadmap-timeline.md`.
- Acceptance criteria are the source of truth for QA — they map directly to test cases in `10-test-plan.md`.

Story format:

```
As a <role>, I want <capability>, so that <value>.
```
